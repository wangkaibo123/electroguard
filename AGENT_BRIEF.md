# ElectroGuard — AI Agent Project Brief

> This document is intended for AI agents tasked with developing or maintaining this project.
> Last updated: 2026-04-05

## 1. Project Overview

ElectroGuard is a **browser-based roguelike tower defense game** built with React 19 + TypeScript + Canvas 2D.
Players place electrical machines on a grid, wire them together to form a power network, and defend a central Core against waves of enemies. After each wave, players choose 1 of 3 random upgrades (towers or wires) to strengthen their defense.

**Genre:** Roguelike tower defense with power network management
**Rendering:** HTML5 Canvas 2D (no game engine framework)
**UI Layer:** React + Tailwind CSS v4
**Build Tool:** Vite 6

## 2. File Structure

```
src/
├── main.tsx              # React entry point (StrictMode → <App />)
├── App.tsx               # Top-level UI: stats bar, canvas, inventory panel, overlays (pick/pause/gameover)
├── index.css             # Tailwind CSS import
└── game/
    ├── types.ts          # All type definitions, constants, TOWER_STATS, PickOption
    ├── engine.ts         # Pure game logic: pathfinding, power grid, spawning, collision, pick generation
    ├── renderer.ts       # Canvas 2D rendering (stateless — takes GameState, draws frame)
    └── useGameLoop.ts    # React hook: game loop, state management, input handling
```

## 3. Architecture & Data Flow

```
User Input (mouse events on <canvas>)
    ↓
useGameLoop (React hook)
    ├── Mutates stateRef.current (GameState)
    ├── Calls engine.ts functions for logic
    ├── Calls sync() to trigger React re-render (UI only)
    └── Calls renderGame() each frame (Canvas rendering)
```

**Key design decisions:**

- **No resource/currency economy** — towers and wires come exclusively from roguelike picks.
- **GameState is mutated directly** via `stateRef`, not through React immutable updates. React state is only used to trigger UI re-renders.
- **Interaction state lives in refs** to keep `gameLoop` callback stable with zero dependencies.
- `**towerMap`** (Map<string, Tower>) maintained for O(1) lookups. Must be rebuilt via `rebuildTowerMap()` after tower add/remove.
- **Rendering is fully stateless** — `renderGame()` reads GameState and draws everything from scratch each frame.

## 4. Roguelike Pick System

### Flow

1. **Game start** → Core pre-placed, inventory: 1 Blaster + 1 Generator + 5 Wires → first pick shown
2. **Player picks 1 of 3** → item(s) added to inventory → status='playing' → 5s countdown
3. **Wave spawns** → enemies fight → all enemies killed → **pick overlay shown** (wave clear bonus scored)
4. **Player picks** → back to step 3 with next wave

### Implementation

- `GameState.needsPick: boolean` — set `true` when wave starts; triggers pick when wave clears
- `GameState.pickOptions: PickOption[]` — 3 random options generated by `generatePickOptions()`
- `GameState.towerInventory: Record<string, number>` — per-type tower counts
- Pick pool defined in `PICK_POOL` constant in `engine.ts` (9 options, 3 drawn each time)
- `handlePick(optionId)` in useGameLoop applies the selection and transitions to playing

### Pick Pool


| Option     | Kind  | Count | Description       |
| ---------- | ----- | ----- | ----------------- |
| Blaster    | tower | 1     | Single turret     |
| Blaster x2 | tower | 2     | Two turrets       |
| Generator  | tower | 1     | Power source      |
| Wall x3    | tower | 3     | Blockers          |
| Wall x5    | tower | 5     | More blockers     |
| Shield     | tower | 1     | Protective bubble |
| Battery    | tower | 1     | Power storage     |
| Wire x3    | wire  | 3     | Connectors        |
| Wire x5    | wire  | 5     | Large bundle      |


## 5. Tower Types (defined in `TOWER_STATS`)


| Type      | Size | Ports    | Key Behavior                                                           |
| --------- | ---- | -------- | ---------------------------------------------------------------------- |
| Core      | 3x3  | 4 output | Central hub. Generates power. Has shield (500 HP, 100px radius).       |
| Blaster   | 2x2  | 2 input  | Auto-fires at enemies in 150px range. 50 dmg. Requires 4 stored power. |
| Generator | 2x2  | 2 output | Power source. Dispatches energy pulses through the network.            |
| Wall      | 1x1  | 0        | 500 HP blocker. No ports.                                              |
| Shield    | 1x1  | 2 input  | Projects 60px protective bubble. Consumes power to recharge.           |
| Battery   | 2x1  | 1in+1out | Stores 4 power, discharges rapidly (100ms).                            |


**All non-wall/non-core/non-battery towers get exactly 2 randomly-directed ports.**

## 6. Power Network

- Towers connect via **Wires** (A* pathfinded grid paths between ports).
- Ports: **output** (amber triangles) → **input** (blue squares). Power flows directionally.
- `updatePowerGrid()` BFS from Core/Generators marks reachable towers as `powered`.
- `dispatchPulse()` BFS through output ports finds nearest tower needing power, sends animated pulse.

## 7. Enemies

- Spawn from random map edges. Count: `2 + wave`. HP: `20 * (1 + wave * 0.2)`.
- Target priority: shields > towers > wires (by nearest distance).
- Attack range: 10px, cooldown: 1000ms.
- Kill reward: +10 score (no resource reward).
- Wave clear bonus: `wave * 20` score.

## 8. Key Engine Functions (engine.ts)


| Function                                     | Purpose                                                               |
| -------------------------------------------- | --------------------------------------------------------------------- |
| `findWirePath(start, end, state)`            | A* pathfinding on grid. Returns cell path or null.                    |
| `updatePowerGrid(state)`                     | BFS to mark all towers reachable from power sources.                  |
| `dispatchPulse(state, source)`               | BFS to find nearest tower needing power, creates animated pulse.      |
| `applyTowerRotation(tower, new, old, state)` | Rotates tower 90deg, swaps dimensions, re-paths wires.                |
| `canPlace(x, y, type, state)`                | Validates placement (bounds, collision, **inventory count > 0**).     |
| `generatePickOptions()`                      | Returns 3 random PickOptions from PICK_POOL.                          |
| `generatePorts(portType)`                    | Creates 2 random-direction ports of given type.                       |
| `repathConnectedWires(state, towerId)`       | Re-runs A* for all wires connected to a moved/rotated tower.          |
| `rebuildTowerMap(state)`                     | Rebuilds `state.towerMap` from `state.towers`. Call after add/remove. |


## 9. Input Handling (useGameLoop.ts)

All mouse events go through three handlers: `mouseDown`, `mouseMove`, `mouseUp`.

**Interaction modes:**

1. **Port click** → Wire drag (creates/replaces wire connection, consumes wireInventory)
2. **Tower click** → Tower drag (repositions, auto-re-paths wires)
3. **Tower click release without drag** → Rotation mode (shows knob UI)
4. **Rotation knob drag** → Live rotation preview, snaps to 90deg on release
5. **Empty grid click with tower selected** → Tower placement (consumes towerInventory)

## 10. Development Guidelines

### Adding a New Tower Type

1. Add the type string to `TowerType` union in `types.ts`.
2. Add stats entry to `TOWER_STATS`.
3. Define port generation logic in `handleCanvasMouseDown` (placement section).
4. Add visual rendering in `drawTowerDetails()` in `renderer.ts`.
5. Add behavior logic in the game loop (power/shield/attack section).
6. Add UI button in `App.tsx` sidebar.
7. Add entries to `PICK_POOL` in `engine.ts`.

### Adding a New Pick Option Type

1. Add to `PICK_POOL` in `engine.ts`.
2. If it's a new `kind` (not 'tower' or 'wire'), update `handlePick()` in `useGameLoop.ts`.
3. Update pick card rendering in `App.tsx` (icon, color).

### Adding a New Enemy Type

1. Extend `Enemy` interface with a `type` field.
2. Modify `spawnEnemy()` to randomize type.
3. Adjust enemy AI section in game loop.
4. Add visual rendering in the enemies section of `renderer.ts`.

### Common Pitfalls

- After modifying `state.towers` (push/filter), always call `rebuildTowerMap(state)`.
- After modifying `state.wires`, always call `updatePowerGrid(state)`.
- `tower.rotation` is visual-only for the knob animation. After applying, it resets to 0.
- `towerInventory` is checked by `canPlace()` — no resources/cost system exists.
- `needsPick` flag drives the pick/wave cycle. Set true when spawning wave, consumed when pick overlay shows.
